# 将 noloong app 作为主交互客户端

Noloong 的桌面 app 不再只是 profile 配置工具，而是默认面向用户的主交互客户端：启动后优先进入 Chat 画布，并默认内嵌 interaction runtime；连接外部 runtime 保留为高级模式。GUI 只消费 agent 会话的展示事件来渲染 transcript、流式回复、运行活动和审批状态；如果现有展示事件缺少语义，就扩展展示事件，而不是让 GUI 读取原始事件或维护第二套事实来源。

这个选择牺牲了“配置工具很薄、运行时完全外置”的简单边界，但换来打开即用的体验、跨 Telegram/微信/桌面一致的会话语义，以及后续把桌面端做成主要工作入口的空间。参考视频只作为 Noloong-native 高保真体验目标：复刻窗口节奏、输入区手感和流式尾迹，不复制外部产品的品牌、文案或协议模型。
