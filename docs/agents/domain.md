# Domain Docs

本文件说明工程技能在探索本仓库时如何读取领域文档和架构决策。

## 开始探索前

按需读取这些文档：

- 根目录的 `CONTEXT.md`
- 根目录的 `docs/adr/`

如果这些文件或目录不存在，静默继续，不要因为缺失而阻塞任务，也不要主动要求创建。领域词汇和架构决策会在需要时由 `/grill-with-docs` 等流程逐步补齐。

## 文件布局

本仓库使用 single-context 布局：

```text
/
├── CONTEXT.md
├── docs/adr/
│   ├── 0001-example-decision.md
│   └── 0002-example-decision.md
└── src/
```

## 使用领域词汇

当输出中需要命名领域概念，例如 issue 标题、重构建议、诊断假设或测试名称时，优先使用 `CONTEXT.md` 中定义的术语。

如果需要的概念还没有出现在 `CONTEXT.md` 中，说明可能存在两种情况：

- 当前表达偏离了项目已有语言，应重新考虑命名。
- 这是一个真实的领域语言缺口，应在后续 `/grill-with-docs` 中补齐。

## 标记 ADR 冲突

如果建议或实现会违反已有 ADR，需要明确指出冲突，而不是静默覆盖。

示例：

```text
这与 ADR-0007 中的事件存储决策冲突，但值得重新讨论，因为……
```
